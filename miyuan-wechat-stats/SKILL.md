# 云发单服务运行状态 (miyuan-wechat-stats)

## 🎯 目标
访问 Grafana Dashboard → 抓取4个时间窗口的核心指标 → 多维对比分析 → 输出健康状态报告。

---

## 📋 执行流程

### Step 1：环境检测
```bash
python3 -c "from playwright.sync_api import sync_playwright; print('OK')" 2>/dev/null || echo "MISSING"
```

### Step 2：运行检查脚本
```bash
python3 ~/.openclaw/workspace/skills/wechat-stats/scripts/grafana_checker.py --output /tmp
```

脚本行为：
1. 使用持久化浏览器 profile（`grafana`），Cookie 已保存，**通常无需重新登录**
2. 若未登录，打开登录页等待用户手动登录（60秒），登录后自动保存 Cookie
3. 依次访问 5m / 15m / 30m / 1h 四个时间窗口
4. 提取核心指标（待推送消息、堆积量、失败率、MQ堆积、推送延迟等）
5. 多窗口对比分析，生成健康报告

### Step 3：读取报告
```bash
cat /tmp/wechat_stats_report.txt
```

---

## 📊 分析策略

**以历史基准为参考线，判断当前是否偏离正常，而非固定阈值硬判断。**

| 指标 | 当前来源 | 历史基准 | 告警条件 |
|------|---------|---------|---------|
| 推送失败率 | 近5min | 6h/24h均值 | 🟡 > 基准1.5x 或 >5% / 🔴 > 基准2.5x 或 >15% |
| 推送时延 | 近5min | 6h/24h均值 | 🟡 > 基准1.5x 或 >60s / 🔴 > 基准2.5x 或 >180s |
| 消息堆积 | 近5min | 6h/24h均值 | 🟡 > 基准1.5x 或 >50K / 🔴 > 基准2.5x 或 >100K |
| MQ堆积 | 近5min | 6h/24h均值 | 🟡 > 基准1.5x 或 >500 / 🔴 > 基准2.5x 或 >2000 |
| 推送吞吐突降 | 近5min | 6h均值 | 🟡 当前 < 历史40% |

---

## ⚠️ 核心约束
1. **只读不写** — 仅抓取数据，不对 Grafana 做任何修改
2. **多窗口对比** — 单点数据不能说明趋势，必须4个窗口联合判断
3. **阈值可调** — 脚本中 `THRESHOLDS` 字典可按实际业务调整
